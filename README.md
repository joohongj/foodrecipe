## Project Overview

- **Purpose**: The purpose of this project is to provide an interactive platform for users to explore the nutritional composition of various foods. By leveraging a professional database, we aim to bridge the gap between complex data and consumer-friendly visualization.
- **Live Data Set**: We used the **USDA FoodData Central API**, which provides authoritative information on food components. By using this project, we hope users learn about the vast variety of food categories available and gain a clearer understanding of how different brands and categories compare in the food industry.

## Features

- **Dynamic Search**: Users can input any food item into a search bar to fetch real-time nutritional data from the USDA database.
- **Responsive Layout**: The data is presented in a flexible grid system that adapts to the user's screen size, ensuring a clear view of food categories and descriptions.
- **Interactive UI Animations**: The site features a custom "wiggle" logo animation and a "flashy" advertisement switcher to enhance user engagement.

## Technologies Used

- **Languages**: HTML, CSS, JavaScript
- **APIs**: USDA FoodData Central API (FDC) for real-time food nutritional data.
- **Tools**: GitHub Pages for hosting, VS Code for development, and Google Fonts for typography.
- **Design**: Figma for initial UI layout and Adobe Stock for visual assets.

## Future Enhancements

- **Nutrient Filtering**: Adding the ability to sort results by specific nutrients (e.g., high protein, low sugar).
- **Comparison Tool**: Allowing users to select two food items to view their nutritional facts side-by-side.
- **Saved Recipes**: Integrating a "Favorite" feature to save specific food items to a local dashboard.
